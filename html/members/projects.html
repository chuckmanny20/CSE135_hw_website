Kaggle Competition for Predicting Traveling Car's Path

Achieved 5th place in the competition of 50 teams with a RMSE of 2.10

One thing we did was use the relative positions of each car instead of their absolute positions. We took the agent car's position and subtracted it from every other car's position to get their position relative to the agent. This helped us train better because it is way easier/common to predict the agent's movement using distances of 5 instead of predicting between points 550 and 555 then 3,000 and 3,005, etc. Also, we only took into account the closest 6 cars because this sped up training, removed the errors in our data ( zeroes for cars that don't exist), and reduced noise. Our input to our final model was a 160 x 7 x 19 x 4 after pre-processing. This represented the first 19 position and velocity time steps of the target agent + up to 6 closest cars within 94 units of the target agent. The resulting output was a 160 x 30 x 2 which represents the target agent's next 30 positions. The loss function we used was RMSE because it gave good estimates to the Kaggle score and works well with problems dealing with distances. We decided on using LSTMs because the input data was a time-series sequence and we wanted to output another time-series sequence so it made sense to use some kind of recurrent neural network. We wanted predictions based on previous predictions and a generative model.
